📈项目总览

本项目是一个跨平台语音 AI 助手，包含 iPhone 端和 AI 服务端：

  • iPhone 端：基于 SwiftUI 的 iOS App，集成 sherpa-onnx 离线语音识别引擎，支持语音转文字、草稿编辑、手动发送、AI 打字指示器
  • AI 服务端：用户自托管的后端服务，支持 Token 认证，接收 {"message": "..."} 并返回 AI 推理结果（最长 120s）

两端通过 HTTP 协议通信，iOS 端使用 130s 超时 URLSession 匹配服务端限制。开源代码中不保留任何服务器地址或 Token，运行时通过 App 界面配置。

💡 信号流核心规律

升级后的系统遵循"音频 → 草稿 → 用户确认 → 网络 → AI 推理 → UI"的受控流动模式。用户在每个关键节点都拥有控制权，同时通过打字指示器和分类错误提示获得完整的反馈闭环。

# FastAPI 示例
@app.post("/send_msg")
async def send_msg(request: Request):
    token = request.headers.get("X-API-Token")
    if token != API_SECRET_TOKEN:
        return {"code": 401, "error": "Invalid token"}
    data = await request.json()
    user_msg = data.get("message", "")
    ai_reply = await call_llm(user_msg)  # 调用大模型
    return {"code": 200, "后端回复": ai_reply}
⚠ 安全提示：开源代码中不硬编码服务器地址和 Token。运行时通过 App 界面输入，或本地维护一个不被 Git 跟踪的配置文件。

🚀下一步扩展路线图

当前系统已实现"语音 → 可编辑草稿 → 手动发送 → AI 推理 → 打字指示器 → 回复气泡"的完整微信式对话体验。以下是值得继续探索的方向：

语音合成反馈 (TTS)
AVSpeechSynthesizer

在 iPhone 端将后端 AI 文字回复通过系统 TTS 朗读出来，实现真正的"语音对话"闭环体验。

对话历史持久化
SwiftData / Core Data

将 messages 保存到本地数据库，支持历史会话查看、搜索、按日期分组，跨启动保留聊天记录。

多轮对话上下文
Conversation Memory

将最近 N 条对话历史一起发送给 AI 服务器，实现上下文感知的连续对话，而非单轮问答。

本地大模型离线运行
llama.cpp / mlx-swift

在 iPhone 本地运行轻量化 LLM（如 Qwen 1.8B、Phi-3），无需网络即可对话，彻底保护隐私。

Markdown / 代码高亮渲染
SwiftUI AttributedString

AI 回复中常包含代码块、公式、列表。将 MessageBubble 升级为支持 Markdown 渲染的富文本组件。
